üìñ How to Use This Tool

Method 1: Automatic Parsing (Recommended for Quick Results)

  1. Upload your FASTA files using drag-and-drop or the browse button
  2. Click "Process Files" to automatically parse sequence identifiers
  3. Download results as CSV or JSON
What this extracts:
  • Species names from sequence IDs (e.g., "G_subdryophilus" ‚Üí "Gymnopilus subdryophilus")
  • Collection numbers and voucher specimens
  • Type specimen status if indicated in ID
  • Basic metadata from sequence naming patterns

Method 2: Manual GenBank Import (For Complete Metadata)

  1. Upload your FASTA file to see the list of GenBank accessions
  2. Copy each accession number (e.g., MN206892)
  3. Go to NCBI Nucleotide Database
  4. Search for the accession and select "GenBank (full)" format
  5. Copy the entire GenBank record and paste it in the text area
  6. Repeat for all sequences or paste multiple records at once
  7. Click "Process GenBank Data"

üîß For Developers: API Access

To enable automatic NCBI fetching, you'll need to set up a backend API. Here's an example:

// Vercel API Route (api/genbank.js) export default async function handler(req, res) { const { accession, email } = req.query; const url = `https://eutils.ncbi.nlm.nih.gov/entrez/eutils/efetch.fcgi?db=nucleotide&id=${accession}&rettype=gb&retmode=text&email=${email}`; const response = await fetch(url); const data = await response.text(); res.setHeader('Access-Control-Allow-Origin', '*'); res.status(200).send(data); }

üìä Output Format

The tool generates CSV and JSON files with the following fields:

  • Fasta ID: Original sequence identifier
  • Species Name: Organism name
  • GenBank Accession: NCBI accession number
  • Location: Collection location
  • Type Status: If it's a type specimen
  • Collection Info: Voucher/specimen information
  • And more... (dates, collectors, references, etc.)
